Projet WEB 2

1 - Itération 1

Voir le repository GitLab du projet

La première partie du projet consistait à :

  • Dans la base de données fournis : changer la clef primaire de la table Utilisateur ainsi que
    les clefs étrangères y faisant réference.
  • étudier les fonctionnalités existantes de l'application fournis.

Ces tâches nous sont adressées sous forme de tickets à résoudres (ticket 1 à 2).

Pour la réalisation du projet je disposais :

-D'une équipe composer de moi et de deux autres camarades
-D'un dépôt distant sur gitLab
-De netBeans comme IDE
-D'un scrum board
-D'un projet inital en Modèle-vue-contrôleur

3 - Itération 3

Voir le repository GitLab du projet

Cette dernierre partie consistait à répondre à l'user story :
En tant qu’utilisateur, je souhaite pouvoir consulter les types de cuisines proposés par chacun des restaurants afin de faciliter ma sélection.

Pour ce faire, la bdd devait être re-penser pour ajouter les types de cuisines et une nouvelle classe métier/DAO écrite pour pouvoir récupérer les données de la bdd.
De plus, les contrôleurs/vue concernés (la liste des resto, le détail d'un reto et la top 4) devaient être modifés pour permettre l'affiche des types de cuisines.
