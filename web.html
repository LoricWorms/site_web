Projet WEB

1 - Création du site web

Voir le repository GitHub du projet

On a commencé par créer le squelette du site suivant les demandes de l'énoncé. On dispose donc d'une page d'accueil permettant d'accèder aux divers services du site.

A cette instant, seules les pages événements, à propos et contact sont opérationnelles.

2 - Le catalogue

Voir le repository GitHub du projet

L'une des premières fonctionnalités que nous avous ajouté est une liste déroulante permettant d'afficher uniquement les livres de la catégorie séléctionnée.

Ecrite en php, cette fonctionnalité intérroge la BDD pour savoir quels livres afficher.

3 - La gestion de la BDD

Voir le repository GitHub du projet

Par la suite nous avons dû concevoir une page CRUD permettant aux "administrateurs" de gérer les utilisateurs et les livres enregistrés dans la BDD.

De plus, une sécurité php a été mise en place pour s'assurer que seul un administrateur connecté puisse y accèder.

4 - Connexion et création de compte

Voir le repository GitHub du projet

Enfin, une page de connexion qui sert notamment à rediriger les utilisateurs "clients" et "administrateurs" vers la bonne page.

Additionnellement, nous avons ajouté une fonction permettant de créer un nouveau compte et de réinitialiser son mot de passe, ce qui a demandé d'adapter le BDD.

Conclusion

Synthèse des compétences :

  • Seul les utilisateurs "administrateur" peuvent accèder au back office (Mettre en place et vérifier les niveaux d’habilitation associés à un service)
  • Création d'un site internet afin de mettre en avnt les services proposés par l'association (Participer à l’évolution d’un site Web exploitant les données de l’organisation.)
